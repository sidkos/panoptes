## Owner-supplied prerequisites (block ONLY the live hosted deploy / GitHub login)

CI + every unit phase is hermetic and needs none of these. Provide them before the
Phase-6/7 `terraform apply` + `helm install` + the live GitHub login:

- A **Panoptes home AWS account** + region + a **dedicated VPC** (SAME account as the
  observed infra — decision #1; never an observed environment's VPC).
- A **GitHub OAuth app**: client id/secret + the **org (and/or team) allowlist** that
  gates access (decision #5).
- A **`panoptes.<domain>`** hostname (Route53 zone or a manual A record); TLS is issued by
  cert-manager + Let's Encrypt (no ACM cert needed — decision #3).
- The per-env **`PanoptesReadRole` ARNs** (from the consumer's IaC, **in-account** — empty
  list is fine for a dev-only deploy; assumed via the IRSA base identity).
- An **SNS topic ARN** + a **Slack webhook URL** for alerts.
- A read-only **in-cluster ClusterRole/SA** (Panoptes' own cluster self-observe) + a
  per-env **kubeconfig/SA token** for any observed cluster the `kubernetes` source reads.

`.env` / Helm values are gitignored / not committed; the GitHub/Slack secrets stay out of
config YAML (`${VAR}` interpolation only).
